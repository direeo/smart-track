# SmartTrack KPI - Render Deployment Guide

## Deployment Status

Your application is configured and ready to deploy on Render.

### Current Configuration

- **Runtime:** Python 3.11.7
- **Framework:** FastAPI + Uvicorn
- **Database:** SQLite (persistent disk)
- **Deployment:** Render free tier

2. **Startup & Error Handling** ✅

   - Health check endpoint at `/health`

3. **Database Configuration** ✅
   - Database path configured via DATABASE_PATH env var
   - Parent directory creation automatic
   - String conversion for Path compatibility

### How to Redeploy

1. Go to **[Render Dashboard](https://render.com)**
2. Select **smarttrack-kpi** service
3. Click **"Redeploy latest commit"**
4. Monitor logs for:
   ```
   ==> Build successful 🎉
   ==> Running 'bash start.sh'
   [OK] Database initialized successfully
   ```

### Troubleshooting Deployment Failures

If the app still exits early:

**Check logs in Render dashboard:**
- Go to service → Logs
- Look for `[ERROR]` messages
- Common issues:
  - Missing template files
  - Missing static directory  
  - Database permission issues

**Manual verification locally:**
```bash
cd models/
python -c "from database import init_db; init_db(); print('✓ DB OK')"
python -c "from app import app; print('✓ App imports OK')"
```

### Environment Variables (Set in Render)

| Variable | Value |
|----------|-------|
| `SESSION_SECRET` | Auto-generated by Render |
| `DATABASE_PATH` | `/opt/render/project/src/models/smarttrack.db` |
| `PYTHONUNBUFFERED` | `true` |

### Persistent Storage

The 1GB disk mounted at `/opt/render/project/src/models` keeps your SQLite database across redeployments. No data loss on redeploy.
